# GitHub 使用指南

## 一、什么是 GitHub

GitHub 是全球最大的代码托管平台，基于 Git 版本控制系统，用于：

  * 代码托管与协作

  * 版本控制管理

  * 开源项目分享

  * 团队协作开发

# 二、快速入门步骤

## 1. 注册与设置

* 访问 github.com 注册账号

* 设置 SSH 密钥（可选）

## 2. 基础操作流程

### 基础工作流

  * 克隆仓库

  * 创建分支

  * 提交更改

  * 推送到远程

  * 拉取更新

## 3. 核心概念

* Repository	仓库，项目存放位置

* Branch	分支，用于并行开发

* Commit	提交，代码变更的快照

* Pull Request	合并请求，代码审查与合并

* Issue	问题跟踪，任务/BUG管理

* Fork	复制他人仓库到自己的账号

## 4. 协作开发流程

* Fork 他人仓库到自己的账户

* Clone 到本地开发

* 创建 Feature Branch 进行修改

* 提交 Pull Request 请求合并

* 参与 Code Review 讨论

* 维护者合并到主分支

## 5. 实用功能
* GitHub Pages：静态网站托管

* Actions：自动化 CI/CD

* Projects：项目管理看板

* Wiki：项目文档

* Discussions：社区讨论

## 三、最佳实践建议
### 良好的提交习惯
* 提交信息清晰明确

* 一次提交一个逻辑修改

* 使用 .gitignore 排除无关文件
## 实战：

* 试着克隆本仓库到你自己的账号上。

* 试着提交一个issue，随便写。

* 试着提交一个文件并删除。

* 试着编写自己的第一个markdown文件。

* 试着查看其他文档。
